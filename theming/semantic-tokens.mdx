---
title: Semantic tokens
description: Understand and extend Fidely UI's semantic token system for foreground, background, border, radii, and shadows.
---

Semantic tokens provide a consistent, theme-aware way to style components across Fidely UI. They abstract raw design values — like `gray.12` — into meaningful names like `fg.default` or `bg.subtle`. These tokens adapt automatically between light and dark modes without any extra work on your part.

<Note>
  Semantic tokens are defined in the `fidelyPreset` and extend Panda CSS's built-in semantic token system. You can further extend them in your own `panda.config.ts`.
</Note>

## Foreground (`fg`)

Use `fg` tokens for text, icons, and interactive label colors.

```ts
fg: {
  default:  { value: '{colors.gray.12}' },
  muted:    { value: '{colors.gray.11}' },
  subtle:   { value: '{colors.gray.10}' },
  disabled: { value: '{colors.gray.9}' },
  error:    { value: '{colors.red.9}' },
}
```

```tsx
<Text color="fg.default">Primary label</Text>
<Text color="fg.muted">Supporting text</Text>
<Text color="fg.subtle">Placeholder or hint</Text>
<Text color="fg.disabled">Disabled label</Text>
<Text color="fg.error">Error message</Text>
```

## Background (`bg`)

Use `bg` tokens for cards, panels, inputs, buttons, and page backgrounds.

```ts
bg: {
  surface:    { value: '{colors.red.1}' },
  default:    { value: '{colors.gray.2}' },
  subtle:     { value: '{colors.gray.3}' },
  muted:      { value: '{colors.gray.4}' },
  emphasized: { value: '{colors.gray.5}' },
  disabled:   { value: '{colors.gray.6}' },
}
```

```tsx
<Box bg="bg.default">Page content</Box>
<Box bg="bg.subtle">Sidebar or panel</Box>
<Box bg="bg.muted">Hover state</Box>
<Box bg="bg.emphasized">Selected or active state</Box>
```

## Border (`border`)

Use `border` tokens for outlines, dividers, focus rings, and card edges.

```ts
border: {
  default:  { value: '{colors.gray.7}' },
  muted:    { value: '{colors.gray.6}' },
  disabled: { value: '{colors.gray.5}' },
  subtle:   { value: '{colors.gray.4}' },
  outline:  { value: '{colors.gray.alpha9}' },
  error:    { value: '{colors.red.9}' },
}
```

```tsx
<Box borderColor="border.default">Default border</Box>
<Box borderColor="border.subtle">Subtle divider</Box>
<Box borderColor="border.error">Error outline</Box>
```

## Radii (`radii`)

Radii tokens define border-radius values for components. The preset generates three semantic radius steps (`s1`, `s2`, `s3`) from the `rounded` option in `fidelyPreset()`.

| Token | Usage |
|---|---|
| `s1` | Small radius (inner elements) |
| `s2` | Medium radius (buttons, inputs) |
| `s3` | Large radius (cards, modals) |

```tsx
<Box borderRadius="s2" />
<Box rounded="md" />   {/* Panda shorthand */}
```

## Shadows

Shadow tokens provide consistent elevation for cards, dropdowns, and modals.

| Token | Description |
|---|---|
| `xs` | Barely visible lift |
| `sm` | Subtle shadow |
| `md` | Card-level elevation |
| `lg` | Dropdown or popover |
| `xl` | Modal-level |
| `2xl` | Full-screen overlay |

```tsx
<Box shadow="sm">Card</Box>
<Box shadow="11px 11px 3px 1px rgba(255, 145, 0, 0.2)">Custom shadow</Box>
<Box shadow="40px -12px var(--shadow-color)" shadowColor="blue">Variable shadow</Box>
```

## CSS variable reference

Every semantic token maps to a CSS custom property generated by Panda CSS:

| CSS variable | Token |
|---|---|
| `var(--colors-fg-default)` | `fg.default` |
| `var(--colors-fg-muted)` | `fg.muted` |
| `var(--colors-fg-subtle)` | `fg.subtle` |
| `var(--colors-fg-disabled)` | `fg.disabled` |
| `var(--colors-fg-error)` | `fg.error` |
| `var(--colors-bg-default)` | `bg.default` |
| `var(--colors-bg-subtle)` | `bg.subtle` |
| `var(--colors-bg-muted)` | `bg.muted` |
| `var(--colors-bg-emphasized)` | `bg.emphasized` |
| `var(--colors-border-default)` | `border.default` |
| `var(--colors-border-subtle)` | `border.subtle` |
| `var(--colors-border-error)` | `border.error` |

## How tokens adapt to dark mode

Fidely UI uses Panda CSS's `_dark` condition. When the `dark` class is applied to the `html` element (via `next-themes` or manually), semantic tokens automatically resolve to their dark-mode values. You never need to write explicit `_dark` overrides when using semantic tokens.

```tsx
{/* This adapts automatically — no _dark needed */}
<Box bg="bg.subtle">
  <Text color="fg.default">Adapts to light and dark</Text>
</Box>
```

## Extending semantic tokens

Add your own semantic tokens in `panda.config.ts` under `theme.extend.semanticTokens`:

```ts panda.config.ts
import { defineConfig } from '@pandacss/dev'

export default defineConfig({
  theme: {
    extend: {
      semanticTokens: {
        colors: {
          bg: {
            DEFAULT:   { value: '{colors.gray.2}' },
            primary:   { value: '{colors.teal.6}' },
            secondary: { value: '{colors.gray.3}' },
          },
          fg: {
            DEFAULT: { value: '{colors.gray.9}' },
            subtle:  { value: '{colors.gray.6}' },
          },
        },
      },
    },
  },
})
```

<Tip>
  You can use raw CSS values (`#fff`, `white`) instead of token references if you need colors that don't change between modes.
</Tip>

## Token naming convention

Semantic tokens follow a `{category}.{scale}` pattern:

```
fg.default       →  colors → fg → default
bg.subtle        →  colors → bg → subtle
border.muted     →  colors → border → muted
```

In CSS they become `var(--colors-{category}-{scale})`, e.g. `var(--colors-fg-default)`.
