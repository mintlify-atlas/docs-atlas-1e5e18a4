---
title: Password input
description: A secure text input with a show/hide visibility toggle, custom icons, and password manager control.
---

`PasswordInput` is a compound component that wraps a text input and a visibility toggle button. It handles the `type` attribute switching between `password` and `text` automatically.

## Import

```tsx
import { PasswordInput } from '@fidely-ui/react'
```

## Basic usage

```tsx
<PasswordInput.Root>
  <PasswordInput.Label>Password</PasswordInput.Label>
  <PasswordInput.Control>
    <PasswordInput.Input />
    <PasswordInput.VisibilityTrigger>
      <PasswordInput.Indicator />
    </PasswordInput.VisibilityTrigger>
  </PasswordInput.Control>
</PasswordInput.Root>
```

## Autocomplete

Use the `autoComplete` prop to signal the browser's autofill intent.

```tsx
{/* For login forms */}
<PasswordInput.Root autoComplete="current-password">
  <PasswordInput.Control>
    <PasswordInput.Input />
    <PasswordInput.VisibilityTrigger>
      <PasswordInput.Indicator />
    </PasswordInput.VisibilityTrigger>
  </PasswordInput.Control>
</PasswordInput.Root>

{/* For registration / change-password forms */}
<PasswordInput.Root autoComplete="new-password">
  <PasswordInput.Control>
    <PasswordInput.Input />
    <PasswordInput.VisibilityTrigger>
      <PasswordInput.Indicator />
    </PasswordInput.VisibilityTrigger>
  </PasswordInput.Control>
</PasswordInput.Root>
```

## Custom icons

Provide custom icons for the visible and hidden states via `PasswordInput.Indicator`.

```tsx
<PasswordInput.Root>
  <PasswordInput.Control>
    <PasswordInput.Input />
    <PasswordInput.VisibilityTrigger>
      <PasswordInput.Indicator
        idleIcon={<EyeOpenIcon />}
        fallbackIcon={<EyeClosedIcon />}
      />
    </PasswordInput.VisibilityTrigger>
  </PasswordInput.Control>
</PasswordInput.Root>
```

## Controlled visibility

Use `visible` and `onVisibilityChange` to drive the show/hide state externally.

```tsx
import { useState } from 'react'

function ControlledPasswordInput() {
  const [visible, setVisible] = useState(false)

  return (
    <PasswordInput.Root
      visible={visible}
      onVisibilityChange={(details) => setVisible(details.visible)}
    >
      <PasswordInput.Control>
        <PasswordInput.Input />
        <PasswordInput.VisibilityTrigger>
          <PasswordInput.Indicator />
        </PasswordInput.VisibilityTrigger>
      </PasswordInput.Control>
    </PasswordInput.Root>
  )
}
```

## Ignore password managers

Set `ignorePasswordManagers` to suppress autofill overlays from password managers. Useful for non-login contexts like API key entry.

```tsx
<PasswordInput.Root ignorePasswordManagers>
  <PasswordInput.Control>
    <PasswordInput.Input placeholder="API key" />
    <PasswordInput.VisibilityTrigger>
      <PasswordInput.Indicator />
    </PasswordInput.VisibilityTrigger>
  </PasswordInput.Control>
</PasswordInput.Root>
```

<Note>
  Password manager suppression currently works for 1Password, LastPass, Bitwarden, Dashlane, and Proton Pass.
</Note>

## Props

### Root

<ParamField body="autoComplete" type="'current-password' | 'new-password'" default="'current-password'">
  Controls the browser's autocomplete behavior.
</ParamField>

<ParamField body="visible" type="boolean">
  Controlled visibility state of the password text.
</ParamField>

<ParamField body="defaultVisible" type="boolean">
  Initial visibility state for uncontrolled usage.
</ParamField>

<ParamField body="onVisibilityChange" type="(details: VisibilityChangeDetails) => void">
  Called when the visibility state changes.
</ParamField>

<ParamField body="disabled" type="boolean">
  Disables the input and the visibility toggle.
</ParamField>

<ParamField body="invalid" type="boolean">
  Marks the input as invalid.
</ParamField>

<ParamField body="required" type="boolean">
  Marks the input as required.
</ParamField>

<ParamField body="readOnly" type="boolean">
  Makes the input read-only.
</ParamField>

<ParamField body="name" type="string">
  The `name` attribute for form submission.
</ParamField>

<ParamField body="ignorePasswordManagers" type="boolean">
  Suppresses autofill overlays from common password managers.
</ParamField>

<ParamField body="asChild" type="boolean">
  Merges root styles onto the child element.
</ParamField>

---

### Indicator

<ParamField body="idleIcon" type="ReactNode">
  Icon shown when the password is hidden (default: eye icon).
</ParamField>

<ParamField body="fallbackIcon" type="ReactNode">
  Icon shown when the password is visible (default: eye-slash icon).
</ParamField>
