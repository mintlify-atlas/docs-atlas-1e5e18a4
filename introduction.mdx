## What is Fidely UI?

Fidely UI is a modern React design system powered by [Ark UI](https://ark-ui.com/) and [Panda CSS](https://panda-css.com/).
It provides accessible, themeable, and composable components for building high-quality web applications.

Rather than focusing only on components, Fidely UI is built around a **design system first** approach,
combining design tokens, styling recipes, and headless primitives into a cohesive and scalable workflow.

The primary goal of Fidely UI is to help teams build consistent, accessible interfaces without sacrificing flexibility or performance.

---

## How Fidely UI is structured

Fidely UI is organized into two layers:

<CardGroup cols={2}>
  <Card title="Panda Preset" icon="palette">
    Design tokens, recipes, and utilities powered by Panda CSS. Can be used independently or as the foundation for your own components.
  </Card>
  <Card title="React Components" icon="code">
    Accessible, headless-first components built on Ark UI and styled using the preset.
  </Card>
</CardGroup>

This separation gives you two usage paths:

- Use **only the preset** with your own components, or
- Use the **full Fidely UI React component library**.

---

## How Fidely UI differs from other UI libraries

Many UI libraries are built on top of **Emotion** or **Tailwind CSS**.

Fidely UI is built on **Panda CSS** — a zero-runtime CSS-in-JS system that generates static, type-safe styles at build time.

<CardGroup cols={2}>
  <Card title="No runtime styling cost" icon="bolt">
    Styles are generated at build time, not injected at runtime.
  </Card>
  <Card title="Faster rendering and smaller bundles" icon="gauge">
    Zero-runtime output means less work for the browser.
  </Card>
  <Card title="Strongly typed design tokens and recipes" icon="shield-check">
    Full TypeScript autocomplete for tokens, variants, and recipes.
  </Card>
  <Card title="Predictable and scalable theming" icon="sliders">
    No hidden abstractions — you own and control the generated styles.
  </Card>
</CardGroup>

Combined with **Ark UI's** headless, accessible primitives, Fidely UI delivers both performance and accessibility out of the box.

<Note>
  This isn't about which library is "better" — every library has its own trade-offs. Fidely UI is optimized for teams that want zero-runtime styling, strong type safety, and composable primitives.
</Note>
